# Контракты данных для dashboard

## Структура папок

- `data/raw` - исходные выгрузки без изменений.
- `data/processed` - нормализованные промежуточные таблицы.
- `data/archive` - исторические файлы и версии выгрузок.
- `data/dashboards` - итоговые Excel-файлы dashboard.
- `templates` - эталонные шаблоны, справочники и маппинги.

## Правила именования файлов

- Имя файла должно содержать источник, период и дату выгрузки.
- Для исторических файлов лучше использовать стабильный шаблон с периодом.
- Временные файлы нужно хранить в `data/raw`, а после обработки переносить в `data/archive`.

## Входные данные

### Bitrix

- Обязательные колонки: `Дата создания`, `Образовательная программа`.
- Допустимые форматы: `xls`, `xlsx`, `csv`.
- Дата должна корректно парситься в день-месяц-год.

### Портал

- Обязательные колонки: `Дата создания`, `Образовательная программа`.
- Допустимые форматы: `xls`, `xlsx`.

### АСАВ

- Обязательные колонки: `Конкурс в магистратуру`, `Договор на обучение`, `Оплата первого периода`, `Приказ о зачислении`.
- Поля дат нормализуются в `applications_dates` и `contracts_dates`.

### АИС ПК

- Для заявлений нужны `Дата` и `Конкурсная группа`.
- Для договоров нужны `Дата заключения` и `Образовательная программа`.
- Для зачислений нужен столбец `Образовательная программа`.

## Метрики

- `applications`, `contracts`, `payments`, `enrollments` считаются по магистратуре через АСАВ и по бакалавриату через АИС ПК.
- `leads` считаются по выгрузкам Bitrix и портала.
- Отдельной метрики `early_invitation` в реестре метрик нет.

## Подготовка к Bitrix API

- Сырые ответы API нужно сохранять отдельно от нормализованных таблиц.
- Поля API следует маппить в те же канонические колонки, которые используются для файловых выгрузок.
- Исторические снимки нельзя перетирать при повторных загрузках.
